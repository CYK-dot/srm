# SRM Overview

## SRM 是什么

SRM（Static Resource Manager）是一套用于嵌入式系统的**静态资源管理工具链**。

它通过声明式 JSON 文件定义资源布局，自动生成高效的 C 语言代码，将资源的**定义、布局、访问**三个环节解耦。

## 解决什么问题

在嵌入式开发中，多个模块需要共享和访问静态资源（配置数据、常量表、计数器等）。传统方式存在以下问题：

| 问题 | 传统方式 | SRM 方式 |
|------|---------|---------|
| 资源分散 | 各模块自行定义，全局变量散落各处 | 集中声明，自动生成 |
| 命名冲突 | 手动保证，容易冲突 | 全局唯一性校验 |
| 偏移计算 | 手动计算，容易出错 | 自动生成，零错误 |
| 内存布局 | 运行时动态分配 | 编译时静态规划 |
| 跨模块共享 | 通过 extern 共享头文件 | 声明式引用，自动生成 |

## 核心价值

**声明式定义**：只需描述资源"是什么"，SRM 自动处理"怎么放"和"怎么访问"。

**编译时保障**：所有校验在构建阶段完成，运行时零开销。

**零动态分配**：完全静态布局，适合资源受限的嵌入式系统。

## 典型场景

- **日志系统**：将日志字符串集中管理，按级别分组，支持编译时裁剪
- **配置管理**：多个模块共享配置项，统一布局
- **版本信息**：从文件加载版本信息，自动生成只读数据
- **计数器池**：多个模块共享计数器资源
